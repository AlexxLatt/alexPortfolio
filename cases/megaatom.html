📋 СИТУАЦИЯ (Проблема)

🎯 Комплексная задача: Создать с нуля полноценную B2B-платформу для электронных компонентов с глубокой интеграцией стороннего API поставщика (getchips.ru). Помимо критически важного функционала поиска и оформления заказов, требовалось реализовать: загрузку прайс-листов из Excel, динамическое ценообразование с кэшированием курсов ЦБ РФ, адаптивный интерфейс с плавными анимациями взаимодействия, административную панель с экспортами, систему логирования всех операций и многоуровневую защиту от атак.

🔥 Критические требования:

  • Масштабируемость: Поиск 100-150+ артикулов одновременно
  • Скорость: Результаты за 2-4 секунды (не 100-200 секунд)
  • Интеграция с API: Внешний поставщик с задержкой 1-2 секунды на запрос
  • Безопасность: Защита от манипуляций ценой через DevTools/Burp Suite
  • База данных: Поддержка 100+ заказов в день без перегрузки

❌ Проблемы до оптимизации:

Последовательные запросы к API: 100 артикулов × 1.5 сек = 150 секунд ожидания 😱

Запрос к БД при каждой операции с корзиной: 100 пользователей × 50 действий = 50,000 запросов к БД в день 💥

Уязвимость манипуляции ценой: Клиент открывает DevTools, меняет цену в коде, добавляет товар за 10₽ вместо 1000₽ 💸

⚙️ РЕШЕНИЕ: Мульти-запросы cURL для параллельной обработки API

✅ Подход:

Вместо ожидания каждого ответа от API по очереди, открываем 100+ соединений одновременно и собираем все результаты сразу.

Примеры кода: Оптимизация производительности

Асинхронный поиск товаров (Multi-cURL)

Задача: Опрос нескольких API-методов одновременно. Переход от последовательных запросов к curl_multi сократил время ожидания ответа с 10-15 секунд до 1-2 секунд. 

                    
                        // ============================================================
                            // ПАРАЛЛЕЛЬНОЕ ВЫПОЛНЕНИЕ HTTP-ЗАПРОСОВ К API
                            // ============================================================
                            function fetch_products_parallel($articleToQty) {
                            $mh = curl_multi_init();
                            $channels = [];
                            // ШАГ 1: Инициализация потоков
                            foreach ($articleToQty as $article => $qty) {
                                $url = 'https://api.getchips.ru/v1/search?' . http_build_query([
                                    'input' => $article,
                                    'token' => CHIP_API_TOKEN
                                ]);
                                $ch = curl_init();
                                curl_setopt_array($ch, [
                                    CURLOPT_URL => $url,
                                    CURLOPT_RETURNTRANSFER => true,
                                    CURLOPT_TIMEOUT => 50,
                                    CURLOPT_SSL_VERIFYPEER => true
                                ]);
                                curl_multi_add_handle($mh, $ch);
                                $channels[$article] = $ch;
                            }
                            // ШАГ 2: Одновременное выполнение всех запросов
                            $running = null;
                            do {
                                curl_multi_exec($mh, $running);
                                if ($running) curl_multi_select($mh, 0.1);
                            } while ($running > 0);
                            // ШАГ 3: Сбор и валидация результатов
                            $allProducts = [];
                            foreach ($channels as $article => $ch) {
                                $response = curl_multi_getcontent($ch);
                                $httpCode = curl_getinfo($ch, CURLINFO_HTTP_CODE);
                                curl_multi_remove_handle($mh, $ch);
                                curl_close($ch);
                                if ($httpCode === 200 && $response) {
                                    $data = json_decode($response, true);
                                    // Обработка данных и сохранение в общий массив...
                                }
                            }
                            curl_multi_close($mh);
                            return $allProducts;
                        }

2. Двойная валидация (Сессия + Проверка цены)

Подход: Исключение возможности подмены цены пользователем через DevTools. Система сверяет входящие данные с кэшированными результатами API из сессии сервера. 

                    
                        // ============================================================
                        // ЗАЩИЩЕННАЯ ОБРАБОТКА ДОБАВЛЕНИЯ В КОРЗИНУ
                        // ============================================================
                        function handle_add_to_cart_secure() {
                            if (session_status() !== PHP_SESSION_ACTIVE) { session_start(); }
                            $uid = trim($_POST['uid'] ?? '');
                            $user_price = floatval($_POST['price'] ?? 0);
                            // 1. ПРОВЕРКА: Существует ли UID в "белом списке" сессии?
                            if (!isset($_SESSION['valid_product_uids'][$uid])) {
                                wp_send_json_error(['code' => 'INVALID_UID']);
                                return;
                            }
                            // 2. ПРОВЕРКА: Не была ли изменена цена в браузере?
                            $server_price = floatval($_SESSION['valid_product_uids'][$uid]);
                            $price_diff = abs($server_price - $user_price);
                            if ($price_diff > 0.01) {
                                wp_send_json_error(['code' => 'PRICE_MISMATCH']);
                                return;
                            }
                            // 3. СОХРАНЕНИЕ: Используем только проверенную цену сервера
                            $cart[] = [
                                'uid'      => $uid,
                                'price'    => $server_price, // <--- Гарантированная цена
                                'name'     => sanitize_text_field($_POST['name'] ?? '')
                            ];
                        }

3. Шифрование AES-256-CBC + Сжатие данных корзины

Задача: Обход лимита куки (4КБ) для хранения 100+ товаров с обеспечением конфиденциальности данных на стороне клиента. 

                
                    // ============================================================
                    // УРОВНИ ЗАЩИТЫ: GZIP -> AES-256-CBC -> BASE64
                    // ============================================================
                    function save_encrypted_cart($cart) {
                        // 1. СЖАТИЕ: Уменьшаем объем JSON данных на ~70%
                        $json_cart = json_encode($cart, JSON_UNESCAPED_UNICODE);
                        $compressed = gzcompress($json_cart);
                        // 2. ШИФРОВАНИЕ: Генерируем уникальный IV для каждой записи
                        $iv_length = openssl_cipher_iv_length('aes-256-cbc');
                        $iv = openssl_random_pseudo_bytes($iv_length);
                        $encrypted = openssl_encrypt(
                            $compressed,
                            'aes-256-cbc',
                            COOKIE_ENCRYPTION_KEY,
                            0,
                            $iv
                        );
                        // 3. КОДИРОВАНИЕ: Склеиваем IV + Шифротекст для передачи
                        $secure_cart = base64_encode($iv . $encrypted);
                        // 4. БЕЗОПАСНАЯ УСТАНОВКА КУКИ (HttpOnly, Secure, SameSite)
                        setcookie('simple_cart', $secure_cart, [
                            'expires' => time() + (7 * DAY_IN_SECONDS),
                            'secure' => true,
                            'httponly' => true,
                            'samesite' => 'Strict'
                        ]);
                     }

🔑 Объяснение флагов безопасности:

  • secure=true
    Передается ТОЛЬКО по HTTPS. Предотвращает перехват на открытых сетях WiFi.
  • httponly=true
    JavaScript не может получить доступ. Предотвращает XSS атаки.
  • samesite=Strict
    Предотвращает CSRF атаки.
