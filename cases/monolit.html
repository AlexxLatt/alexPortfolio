1С-Битрикс Custom Components Mobile First Map API Custom Slider
Ситуация (Situation)
Крупный строительный холдинг «Монолит» (38 лет на рынке) нуждался в обновлении цифрового присутствия. Основная сложность: совместить богатый медиа-контент (слайдеры, карты, формы) с приемлемой скоростью загрузки на CMS 1С-Битрикс, которая известна своей ресурсоемкостью.
Задача (Task)
Реализовать дизайн-код и функционал, обеспечив баланс между гибкостью управления и скоростью работы.
  • Интеграция кастомного слайдера и интерактивных блоков.
  • Настройка работы с картами и веб-формами.
  • Оптимизация: Снижение нагрузки на сервер там, где это возможно.
Действия (Action)
  • Гибридная верстка: Статические блоки (HTML) для контента, который редко меняется — это снижает нагрузку на PHP-процессор.
  • Кастомный JS: Написание легкого слайдера без тяжелых внешних библиотек.
  • Интеграция: Подключение API карт и форм через стандартные компоненты Битрикса для удобства администрирования.
Результат (Result)
  • Сайт стабилен под нагрузкой благодаря кэшированию.
  • Контент легко редактируется через админку (инфоблоки, слайдеры).
  • Реализована полная адаптивность под мобильные устройства.
  • Удалось избежать «раздувания» кода там, где динамический функционал не требовался.

Почему эти 2 страницы?

Эти страницы демонстрируют ключевой функционал проекта: «Главная» показывает работу с продукцией в виде табов, видео-контентом и кастомной адаптацией блоков. «О компании» демонстрирует интеграцию слайдера с 1С-Битрикс, интерактивные карты, формы обратной связи и сложные информационные блоки. Обе страницы полностью адаптивны и оптимизированы для быстрой загрузки.

Технические особенности

Главная страница: Модульная архитектура

Главная страница реализована по модульному принципу через bitrix:main.include. Это позволяет контент-менеджерам редактировать отдельные блоки через административную панель без вмешательства в код шаблона.

  • Гибкое управление: Каждый блок — отдельный include-файл в /include/mainpage/
  • Кэширование: Агрессивное кэширование компонентов (CACHE_TIME => 36000000)
  • Композитный режим: COMPOSITE_FRAME_MODE => "A" для динамической подгрузки
  • Видео-секция: Интеграция через bitrix:news.detail с кастомным шаблоном
  • Адаптивные слайдеры: Кастомная верстка без тяжелых библиотек
<!-- Верхний слайдер (редактируемая область) --> <?php $APPLICATION->IncludeComponent( "bitrix:main.include", ".default", [ "AREA_FILE_SHOW" => "file", "PATH" => SITE_DIR . "include/mainpage/comp_banners_top_slider.php", "EDIT_TEMPLATE" => "standard.php", "CACHE_TYPE" => "A", "CACHE_TIME" => "36000000" ] ); ?> <!-- Видео-секция через news.detail --> <?php $APPLICATION->IncludeComponent( "bitrix:news.detail", "main_video_template", [ "ELEMENT_ID" => "33307", "IBLOCK_ID" => "43", "PROPERTY_CODE" => ["VIDEO_URL", "DESCRIPTION"], "CACHE_TYPE" => "A", "COMPOSITE_FRAME_MODE" => "A" ] ); ?> <!-- Новости с кастомным шаблоном --> <?php $APPLICATION->IncludeComponent( "bitrix:news.list", "main_news", [ "IBLOCK_ID" => "12", "NEWS_COUNT" => "3", "CACHE_TIME" => "36000000", "SORT_BY1" => "ACTIVE_FROM", "SORT_ORDER1" => "DESC" ], false ); ?>
Страница «О компании»: Слайдер & Компоненты

Вместо тяжелых библиотек был реализован легкий слайдер на чистом JS с интеграцией компонента aspro:com.banners.optimus. Это позволило выводить динамический контент из инфоблоков без потери производительности.

// Пример вызова компонента с кэшированием $APPLICATION->IncludeComponent( "aspro:com.banners.optimus", "company_slider", [ "IBLOCK_ID" => "38", "CACHE_TYPE" => "A", "CACHE_TIME" => "36000000", "COMPOSITE_FRAME_MODE" => "A" ] );
Интерактивная Карта и Формы

Реализована логика переключения типов карт (Яндекс/Google) через настройки главного модуля. Веб-формы настроены с использованием bitrix:form.result.new с AJAX-обработкой для удобства пользователя.

// Динамический выбор типа карты $arMapTypes = ['YANDEX', 'GOOGLE']; $sMapType = $arMapTypes[\Bitrix\Main\Config\Option::get( "aspro.optimus", "MAP_TYPE", "0", SITE_ID )]; // Включение формы с AJAX $APPLICATION->IncludeComponent( "bitrix:form.result.new", "inline2", [ "WEB_FORM_ID" => "3", "USE_EXTENDED_ERRORS" => "Y" ] );

🛠️ Технический стек

Backend: PHP 8.1+, 1С-Битрикс (Управление сайтом), Bitrix D7 Core, MySQL, Кэширование компонентов

Frontend: Vanilla JavaScript (ES6+), HTML5/CSS3 (BEM), AJAX для веб-форм, интеграция API Яндекс.Карт

Безопасность: Встроенный проактивный фильтр (WAF) Битрикса, SSL-сертификация, разделение прав доступа (Администратор/Контент-менеджер), защита форм от спама (CAPTCHA/reCAPTCHA)
