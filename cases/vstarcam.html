VSTARCAM: Система подписок

Платежный шлюз PayKeeper + WooCommerce + Автопродление + ФЗ-54
PHP 8.2+ WordPress/WooCommerce PayKeeper API ФЗ-54 (Фискализация) QR Codes (CPT) HMAC-SHA256 / MD5 MySQL + wp_options
Ситуация (Situation)
Сайт vstarcam.ru (продажа камер видеонаблюдения) нуждался в автоматизированной системе продажи подписок на облачный сервис. Требовалась интеграция с PayKeeper, мгновенная доставка QR-кодов активации на email и поддержка фискальных чеков по ФЗ-54.
Задача (Task)
  • Разработка кастомного шлюза WC_PK_Gateway для WooCommerce
  • Реализация фискальной корзины (PaykeeperPayment) с корректным расчетом налогов
  • Обработка webhook'ов с проверкой цифровой подписи
  • Система выдачи QR-кодов (CPT qr_code) после оплаты
  • Логирование всех транзакций и поддержка возвратов
Действия (Action)
  • Реализовал класс WC_PK_Gateway: process_payment(), pk_handler(), process_refund()
  • Вынес логику ФЗ-54 в отдельный класс PaykeeperPayment (setOrderParams(), correctPrecision())
  • Настроил подпись форм: SHA256 для отправки, MD5 для проверки callback
  • Создал paykeeper_notify.php для парсинга orderid и автоматической отправки QR
  • Реализовал логирование через pk_write_log() с версией плагина/WP/WooCommerce
Результат (Result)
  • Полностью автоматизированный цикл: оплата → фискализация → выдача QR → email
  • Корректная работа с возвратами через API PayKeeper (/change/payment/reverse/)
  • Поддержка разных типов покупателей (fiz/ur) через парсинг orderid
  • Система логирования для быстрой отладки инцидентов
  • Готовая архитектура для масштабирования на другие тарифы

Архитектура решения

Поток данных (Payment Flow):

1
WC_PK_Gateway::process_payment(): Формирование заказа, расчет fiscal_cart через PaykeeperPayment
2
pk_form() → pk_get_payment_form(): Генерация HTML-формы с подписью SHA256($sum.$clientid.$orderid.$secret)
3
Оплата на стороне PayKeeper: Клиент вводит данные карты, PayKeeper обрабатывает транзакцию
4
Callback на wc_pk_gateway: PayKeeper шлет POST → WC_PK_Gateway::pk_handler()
5
Проверка подписи: md5($id.$sum.$clientid.$orderid.$secret) → при успехе payment_complete()
6
paykeeper_notify.php: Парсинг orderid, поиск QR (CPT), отправка email через wp_mail()
7
Финализация: update_post_meta('_qr_status', 'paid'), wc_reduce_stock_levels(), лог в файл

Примеры кода из проекта

1. Проверка подписи callback (WC_PK_Gateway::pk_handler)

// ============================================
// ПРОВЕРКА ЦИФРОВОЙ ПОДПИСИ ОТ PAYKEEPER
// ============================================

public function pk_handler() {
  // Получаем данные из POST-запроса
  $id         = $_POST['id'] ?? '';
  $sum        = $_POST['sum'] ?? '';
  $clientid   = $_POST['clientid'] ?? '';
  $order_id   = $_POST['orderid'] ?? '';
  $sign       = $_POST['sign'] ?? '';
  
  // Формируем строку для хэша по документации PayKeeper
  $string_to_sign = $id . $sum . $clientid . $order_id . $this->secret;
  $calculated_sign = md5($string_to_sign);
  
  // Сравниваем подписи (защита от поддельных запросов)
  if (!hash_equals($calculated_sign, $sign)) {
    $this->pk_write_log('Invalid signature received');
    wp_die('Access denied', 'Error', ['response' => 403]);
  }
  
  // Подпись верна — обрабатываем оплату
  $order = wc_get_order($order_id);
  if ($order && $order->has_status(['pending', 'on-hold'])) {
    $order->set_transaction_id($id);
    $order->payment_complete();
    $this->pk_write_log("Order {$order_id} completed");
  }
}

2. Парсинг orderid и выдача QR (paykeeper_notify.php)

// ============================================
// ПАРСИНГ ORDERID И ВЫДАЧА QR-КОДА
// Формат: fiz_2024_user123_PRO_3
// ============================================

// Разбиваем orderid на части
$parts = explode('_', $payment['orderid']);
$buyer_type_code = $parts[0] ?? 'fiz';  // 'fiz' или 'ur'
$tarif_code      = $parts[3] ?? '1';    // 1..5 → тариф
$count           = intval($parts[4] ?? 1); // количество месяцев

// Маппинг кодов тарифов в человекочитаемые названия
$tarif_map = [
  '1' => 'Базовый',
  '2' => 'Стандарт',
  '3' => 'Про',
  // ...
];

// Сохраняем подписку в wp_options (для MVP)
$subscriptions = get_option('custom_subscriptions_data', []);
$subscriptions[] = [
  'date'        => current_time('mysql'),
  'order_id'    => $payment['orderid'],
  'email'       => $payment['client_email'],
  'tariff'      => $tarif_map[$tarif_code] ?? 'Unknown',
  'person_type' => $buyer_type_code === 'ur' ? 'Юр. лицо' : 'Физ. лицо',
];
update_option('custom_subscriptions_data', $subscriptions);

// Находим неплаченный QR для данного тарифа (CPT qr_code)
$qr_query = new WP_Query([
  'post_type'      => 'qr_code',
  'meta_query'     => [
    ['key' => '_tariff_type', 'value' => $tarif_code],
    ['key' => '_qr_status',   'value' => 'unpaid'],
  ],
  'posts_per_page' => 1,
  'orderby'        => 'date',
  'order'          => 'ASC',
]);

if ($qr_query->have_posts()) {
  $qr_query->the_post();
  $post_id  = get_the_ID();
  $qr_value = get_post_meta($post_id, '_qr_value', true);
  
  // Отправляем email с QR-кодом
  $this->send_qr_email($payment['client_email'], $qr_value, $tarif_map[$tarif_code]);
  
  // Помечаем QR как оплаченный
  update_post_meta($post_id, '_qr_status', 'paid');
  setcookie('qr_sent_' . $post_id, '1', time() + 86400); // защита от дублей
  
  // Обновляем статус WooCommerce заказа
  $this->update_woocommerce_order_status($payment['orderid'], 'completed');
}

3. Фискальная корзина ФЗ-54 (PaykeeperPayment)

// ============================================
// РАСЧЁТ ПОЗИЦИЙ ДЛЯ ФИСКАЛЬНОГО ЧЕКА (ФЗ-54)
// paykeeper.class.php
// ============================================

class PaykeeperPayment {
  
  public function setOrderParams($order) {
    $this->order = $order;
    $this->buildFiscalCart();
    return $this;
  }
  
  private function buildFiscalCart() {
    $cart = [];
    
    // Товары из заказа
    foreach ($this->order->get_items() as $item) {
      $cart[] = [
        'name'     => $item->get_name(),
        'price'    => $this->correctPrecision($item->get_total()),
        'quantity' => $item->get_quantity(),
        'sum'      => $this->correctPrecision($item->get_total() * $item->get_quantity()),
        'tax'      => $this->mapWooTaxTo54($item->get_tax_class()),
      ];
    }
    
    // Доставка (если есть)
    if ($this->order->get_shipping_total() > 0) {
      $cart[] = [
        'name'     => 'Доставка',
        'price'    => $this->correctPrecision($this->order->get_shipping_total()),
        'quantity' => 1,
        'sum'      => $this->correctPrecision($this->order->get_shipping_total()),
        'tax'      => 'vat20',
      ];
    }
    
    // Применяем скидки и корректируем точность до копеек
    $this->fiscal_cart = $this->applyDiscounts($cart);
    $this->fiscal_cart = $this->correctPrecision($this->fiscal_cart);
  }
  
  // Корректировка точности: округление до 2 знаков
  private function correctPrecision($value) {
    return round(floatval($value), 2, PHP_ROUND_HALF_UP);
  }
  
  public function getFiscalCartEncoded() {
    // Кодируем корзину в base64 для отправки в PayKeeper
    return base64_encode(json_encode($this->fiscal_cart, JSON_UNESCAPED_UNICODE));
  }
}

4. Логирование операций (pk_write_log)

// ============================================
// ЗАПИСЬ ЛОГА С ИНФОРМАЦИЕЙ О СРЕДЕ
// ============================================

private function pk_write_log($message) {
  $log_entry = sprintf(
    "[%s] PK v%s | WP %s | WC %s | PHP %s | %s\n",
    current_time('mysql'),
    PK_PLUGIN_VERSION,
    $GLOBALS['wp_version'],
    WC_VERSION,
    PHP_VERSION,
    $message
  );
  
  $log_file = PK_PLUGIN_DIR . '/logs/paykeeper-' . date('Y-m-d') . '.log';
  
  // Создаём директорию логов, если нет
  if (!file_exists(dirname($log_file))) {
    wp_mkdir_p(dirname($log_file));
  }
  
  // Записываем в файл (FILE_APPEND для добавления)
  file_put_contents($log_file, $log_entry, FILE_APPEND | LOCK_EX);
}

🛠️ Технический стек

Backend: PHP 8.2+, WordPress (WooCommerce API), PayKeeper SDK/REST API, PHPOffice/PhpSpreadsheet (обработка реестров QR), cURL (взаимодействие с эквайрингом)

Frontend: JavaScript (ES6+), WooCommerce Checkout Hooks, AJAX/Fetch для валидации статусов, адаптивные Email-шаблоны (HTML/CSS)

Инфраструктура: MySQL (Custom Post Types для QR-кодов), Cron (проверка просроченных подписок), интеграция с онлайн-кассами (ФЗ-54), логирование в ротируемые файлы

Производительность: Оптимизация поиска по мета-полям, кэширование транзитных данных (WP Transients), минимизация нагрузки на БД при обработке вебхуков
